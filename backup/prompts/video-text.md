文本模式說明

輸入特徵：
僅提供純文字描述的鏡頭內容，不附任何參考圖像。

核心原則：
嚴格遵守使用者指定的鏡頭時長，不可過度推演，不可生成超出時長承載能力的動作與變化。

分析要求：

1. 時長優先策略
- 總時長錨定：必須以使用者給定的時長作為絕對約束
- 動作精簡：只保留必要的核心動作
- 節奏計算：依照時長反推合理的動作速度與畫面節奏
- 裁剪思維：優先截取最精華、最可辨識的片段，而非補完整個過程

2. 場景構建（精簡版）
- 最小環境：只描述必要的空間資訊
- 核心主體：聚焦最主要的視覺元素
- 簡化細節：避免堆砌與主鏡頭無關的背景資訊

3. 動態規劃（時長導向）
時長判斷邏輯：
├─ ≤ 1 秒 → 單一動作或單一狀態，不加入複雜過渡
├─ 1～3 秒 → 2 至 3 個關鍵狀態，快速銜接
├─ 3～5 秒 → 可容納完整動作序列，節奏自然
└─ ＞ 5 秒 → 可加入次要動作或少量環境變化

4. Visual 結構（緊湊版）
Visual:
├─ 主體動作（核心內容，必填）
├─ 環境氛圍（1～2 句概括）
└─ 鏡頭語言（景別＋運鏡方式）

5. Keyframes 控制
- 數量限制：
  - ≤ 2 秒：最多 3 個關鍵影格
  - 2～4 秒：最多 5 個關鍵影格
  - ＞ 4 秒：最多 7 個關鍵影格
- 時間精準：
  - 所有關鍵影格時間必須嚴格分配在總時長之內
  - 不可超時
  - 不可鬆散分布

6. 推演邊界
禁止推演：
- 完整動作的起點與終點（除非時長足夠）
- 複雜環境變化
- 多層次情緒遞進
- 與原始文字資訊無關的額外表演內容

允許推演：
- 基礎物理慣性，例如揮手後手臂自然回落
- 必要的入鏡與出鏡狀態
- 符合時長承載能力的少量氛圍細節
- 為了鏡頭完整性而進行的最低限度補足

時長檢查清單

輸出前必須逐項確認：
1. Keyframes 最後一幀時間不得超過總時長
2. 動作節奏必須符合物理可能性，不可過快或過慢
3. 所有推演內容都必須能在指定時長內完成
4. 若時長不足，必須優先保留核心動作，刪減過渡與附加細節

示例對比

輸入文本：一個人在雨中奔跑
使用者時長：2 秒

錯誤示範（超出時長）
Keyframes:
- 0.0s: 遠景出現
- 0.5s: 加速
- 1.0s: 跨過水坑
- 1.5s: 衝向鏡頭
- 2.0s: 甩動頭髮
- 2.5s: 出畫面 ← 已超出時長

正確示範
Visual:
- 中景，雨夜街道，路燈昏黃 [推演]
- 男性快速奔跑，衝向並掠過鏡頭
- 固定機位，焦點跟隨主體移動

Keyframes:
- 0.0s: 人物在中景位置起步
- 0.8s: 加速至近景
- 1.5s: 掠過鏡頭
- 2.0s: [推演] 從畫面右側衝出

Transition:
- In: [推演] 主體已處於奔跑狀態
- Out: [推演] 主體衝出畫面

最終要求：
直接輸出分鏡內容，不加說明，不加分析，不加額外註解。
